## Key Observations
- Quotes: qfu_quote includes quote number, customer, amount, assigned-to, TSR, CSSR name, next follow-up, overdue since, status, source dates, branch/region/source family, and current-state lifecycle fields.
- Quote lines: qfu_quoteline includes quote number, line number, product/material data, follow-up owner/notes/date, status, CSSR, TSR, branch/region/source lineage, and unique key.
- Branches/regions: qfu_branch and qfu_region store branch/region slugs, names, thresholds, mailbox address, and manager notes.
- TSR/CSSR/managers/users: TSR and CSSR are mostly text fields on quote/line/backorder/exception tables; dedicated staff, manager, role, and assignment tables were not found.
- Alerts/logs: qfu_ingestionbatch is the main freshness/log source. A dedicated alert log table was not found.
- Primary keys: Dataverse row IDs are exposed for each table; operational natural keys rely on qfu_sourceid, qfu_uniquekey, branch code, source family, quote number, sales document, and line fields.
- Duplicate prevention: Current-state docs require qfu_sourceid patterns such as branch|SP830|quotenumber and branch|ZBO|salesdoc|line; scripts audit duplicates but a complete alternate-key solution export was not found.
- Missing relationships: No exported relationship metadata proves branch-to-staff, branch-to-manager, quote-to-assignee, alert-to-quote, or staff-to-role relationships.
- Hardcoded staff/branch data: Branch pilot values 4171, 4172, 4173, Southern Alberta slugs, and branch names are present in source and flow artifacts.
