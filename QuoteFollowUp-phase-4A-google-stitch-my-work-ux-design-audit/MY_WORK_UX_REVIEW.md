# My Work UX Review

The My Work page should be a desktop-first daily workbench, not a generic CRM table.

## Layout

- Header with title, date, branch/team context, refresh timestamp, role context, and freshness status.
- KPI card strip for Due Today, Overdue, Quotes >= $3K, Missing Attempts, Roadblocks, Assignment Issues.
- Priority tabs: Overdue, Due Today, High Value, Needs Attempts, Waiting, Roadblocks, All Open.
- Dense work list with quote, customer placeholder, value, attempts, next follow-up, last followed up, TSR, CSSR, sticky note preview, and next action.
- Right-side quote detail panel.
- Quick action bar.

## Behavior

- Overdue and Due Today must be visually impossible to miss.
- Assignment issues should be visible but separated from normal work.
- Sticky note preview should be visible before opening the record.
- Empty, loading, error, and no-assignment states should be designed as first-class states.
- Mobile quick review is secondary and should not replace the desktop workbench.
